# AI Performance Test Assistant (Fanout)

Этот проект представляет собой **ИИ-ассистента для нагрузочного тестирования**, который автоматически анализирует результаты стресс-тестов микросервисной архитектуры.
Сама архитектура (социальная сеть с лентой и лайками) служит демонстрационной платформой для показа возможностей автоматизированного анализа производительности с помощью LLM.

## Основная идея

Традиционный анализ результатов нагрузочного тестирования требует ручного изучения графиков и метрик. Данный проект автоматизирует этот процесс:
1.  **Генерация нагрузки**: k6 создает реалистичный трафик.
2.  **Сбор данных**: Prometheus и cAdvisor собирают технические метрики сервисов и контейнеров.
3.  **Интеллектуальный анализ**: Локальная языковая модель (через Ollama) получает консолидированные метрики и выдает отчет с описанием аномалий, потенциальных узких мест и рекомендаций.

## Архитектура системы

### 1. Компоненты ИИ-ассистента (`perftest-ai-assistant`)
*   **Ollama**: Сердце системы. Позволяет запускать современные LLM (например, Llama 3) локально без передачи данных во внешние облака.
*   **Скрипт анализа (`run_stress.sh`)**: Связующее звено, которое запускает тесты, опрашивает Prometheus API и формирует промпт для нейросети.
*   **Prometheus**: Хранилище временных рядов с метриками RPS, Latency, CPU/RAM и системных вызовов.
*   **cAdvisor**: Предоставляет детализированную информацию о потреблении ресурсов Docker-контейнерами.

### 2. Демонстрационные микросервисы
*   **Feed Service (`feed_svc`)**: Работает с лентой новостей (Go, PostgreSQL).
*   **Like Service (`like_svc`)**: Управляет лайками (Go, PostgreSQL).
    *Микросервисы специально спроектированы так, чтобы под нагрузкой проявлялись различные паттерны деградации производительности для проверки работы ИИ.*

## Технологический стек

*   **AI/ML**: Ollama
*   **Тестирование**: k6
*   **Мониторинг**: Prometheus, cAdvisor
*   **Бэкенд**: Go (Golang), PostgreSQL
*   **Оркестрация**: Docker Compose

## Быстрый старт

### Требования

*   Docker и Docker Compose
*   k6 (для запуска нагрузочных тестов)
*   curl, jq (для работы скриптов анализа)

### Запуск проекта

1.  Соберите и запустите все сервисы:
    ```bash
    docker-compose up -d --build
    ```

2.  Наполните базу данных начальными данными:
    ```bash
    ./fill_data.sh
    ```

### Нагрузочное тестирование и AI-анализ

В папке `perftest-ai-assistant` находится основной инструмент проекта.

1.  Перейдите в директорию:
    ```bash
    cd perftest-ai-assistant
    ```
2.  Запустите цикл "тест -> анализ":
    ```bash
    ./run_stress.sh
    ```

Скрипт выполнит автоматизированный рабочий процесс:
*   Запустит сценарий k6 для генерации нагрузки.
*   Извлечет ключевые метрики (RPS, Latency, Error Rate) из Prometheus.
*   Получит системные метрики (CPU/RAM usage) из cAdvisor.
*   **Сформирует промпт для локальной LLM**, содержащий все собранные данные.
*   Выведет отчет, сгенерированный ИИ, с глубоким анализом производительности системы.
